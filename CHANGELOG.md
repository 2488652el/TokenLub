# 更新日志

## Unreleased

## 1.0.7 - 2026-07-17

- 新增“异常价格变动需审批”开关，默认开启并持久化用户选择。
- 关闭审批后，待审批价格变更立即放行，后续官方目录同步也会直接应用异常价格变化。
- 手动同步官方价格会根据审批开关决定是否需要确认。

## 1.0.6 - 2026-07-17

- 修复本机 CLI Session 解析触发逻辑：打开 API Keys 页面不再自动解析。
- 自动解析仅在开关开启时由后台调度；手动解析仅通过“解析全部”或来源卡片的“解析入库”执行。
- Dashboard、Provider 汇总和请求日志的刷新操作不再隐式扫描 CLI 日志，关闭自动解析后状态也会持续保持。

## 1.0.5 - 2026-07-16

- 新增应用内检查更新与后台自动更新：从 GitHub Releases 获取当前系统对应版本，显示下载进度，下载完成后静默覆盖安装并重启。
- 更新只替换应用安装目录，SQLite 数据库与本地设置继续保存在 Electron 用户数据目录中。

## 1.0.4 - 2026-07-16

- 云端同步设置支持选择本地备份目录，预览会显示实际备份位置。
- 新增 Ubuntu 一键部署脚本与 Docker Compose 配置，支持 HTTPS 域名和 SSH 隧道两种同步服务模式。
- 增加服务器健康检查、备份、升级和卸载命令，并补充部署验收文档与自动化测试。
- 中文 README 作为默认项目首页，加入产品截图、功能展示和 GitHub Release 下载入口。

## 1.0.3 - 2026-07-15

- Web 控制台新增一键绑定桌面应用，通过 `tokenlub://` 唤起 TokenLub。
- 绑定过程使用两分钟有效且只能兑换一次的票据，不在链接中传递密码或会话令牌。
- 桌面端支持冷启动和已运行状态下接收绑定链接，并自动保存加密同步会话。

## 1.0.2 - 2026-07-14

- 加入设置、模型定价与余额快照的多设备云同步，支持持久化 outbox、分页拉取、bootstrap、冲突处理和设备撤销。
- 加入内置账号登录、设备与会话管理，以及无域名的 Ubuntu/Docker + SSH 隧道自托管部署方案。
- 补齐 PostgreSQL 迁移、历史清理、备份恢复、限流、隐私审计和双设备同步验证。

## 1.0.1 - 2026-07-10

- 正式切换 TokenLub 品牌资源和 Windows 安装产物命名。
- 余额查询页面的卡片头部改为复用 Provider logo，和 API Keys 页面保持一致。
- 清理旧版 TokenScope 安装包，补充中英文 README，并更新发布说明到 1.0.1。

## 1.0.0 - 2026-07-09

- 作为首个正式版本发布，整合用量总览、Provider 汇总、请求日志、价格配置、告警、API Keys 与本机 CLI Session 解析能力。
- 补齐请求日志的服务端筛选与全量 CSV 导出能力，并完成 Electron 打包链路整理，准备发布 Windows 安装包与便携版。

## 0.1.4 - 2026-07-09

- 在 0.1.3 的用量统计 UI 重构基础上，补齐请求日志分页能力，支持按页浏览与按日期跳转到当日请求列表。
- 修复左侧导航在不同页面下的对齐异常，统一图标、文字、角标与高亮底色的纵向节奏。
- Provider 汇总的每日费用趋势改为与用量概览一致的曲线图逻辑，按费用维度展示近 30 天成本波动，并修复原先空白不出图的问题。
- 用量概览的“当日”视图改为按小时展示 Token 消耗趋势，便于观察一天内的峰值分布。
- MiniMax Coding Plan 卡片将 `1.5h 限额` 更正为 `5h 限额`，并接入官方 `token_plan/remains` 配额读取，自动显示 5 小时与周限额。

## 0.1.3 - 2026-07-09

- API Keys 面板接入 LobeHub 模型/厂商 SVG 图标，DeepSeek、LongCat、MiniMax、Claude Code、Codex 等卡片会显示对应品牌标识。
- API Keys 的 Provider 筛选项增加小图标，缺少映射时使用稳定的首字母占位。
- 本机 Session 解析统一放入 API Keys 管理，并支持 Claude Code / Codex 用量卡片与自动解析开关。
- 用量概览支持当日、7 天、30 天、全部时间段，并使用多模型曲线图展示 Token 消耗。
- 消费统计统一按人民币展示，余额快照改为消费统计。
- API Key 卡片按不同 provider / coding plan 显示差异化信息，并增加编辑入口。
